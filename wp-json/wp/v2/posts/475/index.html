<\/p>\n

WhiteSource Bolt + Azure Pipelines<\/p>\n

Till now we have seen how to create Pipelines in Azure DevOps and how to Integrate Sonar Scanning in our Build Pipelines.<\/p>\n

Open Source Libraries can be very handy and can help in achieving the functionality very fast but managing multiple libraries used in our code can be a tedious task. For this problem, there is a tool named WhiteSource Bolt<\/strong> which can be integrated with our Azure DevOps Pipeline<\/strong> and it will generate a report of Open Source Libraries<\/strong> used, the Licenses<\/strong>, and the vulnerabilities in the libraries<\/strong>.<\/p>\n

Integrate WhiteSource in Azure Pipelines<\/strong><\/p>\n

We just need to follow the below small steps and WhiteSource integration will be done:<\/p>\n

1. Install WhiteSource Bolt Plugin from the Marketplace<\/strong><\/p>\n

    \n
  • \n Go to Marketplace and search for WhiteSource bolt<\/li>\n<\/ul>\n

    <\/p>\n

    Just click on Get It Free to install the Extention to your Azure DevOps Organisation.<\/p>\n

    <\/p>\n

    2. Setup WhiteSource Bolt Plugin<\/strong><\/p>\n

      \n
    • \n For setting up the WhiteSource plugin, Go to WhiteSource Tab under Pipelines in any project. ( This tab will be shown only if WhiteSourceBolt Plugin is installed)<\/li>\n<\/ul>\n

      <\/p>\n

        \n
      • \n Just fill the required details and click Get Started.<\/li>\n<\/ul>\n

        <\/p>\n

          \n
        • \n Once you click on Get Started, The below screen will be shown.<\/li>\n<\/ul>\n

          <\/p>\n

          That’s it! WhiteSource has been configured.<\/p>\n

          3. Integrating WhiteSource Scanner into Azure Pipelines<\/strong><\/p>\n

            \n
          • \n Goto Pipeline Definition, and add a task for WhiteSource Bolt in the YAML file.<\/li>\n<\/ul>\n

            <\/p>\n

            I have removed the Sonar Integration from the YAML just for the Demo purpose but it should be present with the WhiteSource Scanner.<\/p>\n

            4. Execute the Pipeline and get the Report<\/strong><\/p>\n

            Once the Pipelines execution is completed, WhiteSource Report will be available under Pipelines -> WhiteSource Tab.<\/p>\n

            <\/p>\n

            This report will show all the Vulnerabilities present in the OpenSource Libraries used in the Project.<\/p>\n

            Also, We can export this report in various formats like HTML, pdf, JSON and excel.<\/p>\n

            <\/p>\n

            Voila!! Our WhiteSource Integration in Azure Pipelines has been done.<\/strong><\/p>\n
